# 3. 入力データと変換仕様

## 入力データ
- ソース URL:
  - `https://www.jpx.co.jp/markets/statistics-equities/misc/tvdivq0000001vg2-att/data_j.xls`

## 目的
- 入力 `data_j.xls` の各行から「証券コード」「17 業種コード」「33 業種コード」を抽出し、
  - `17` 各コードごとに証券コード配列を作る
  - `33` 各コードごとに証券コード配列を作る
- 文字列として JSON 配列に書き出す

## 変換の前提（実装要件）
`data_j.xls` の列名やシート名は JPX の更新で変わり得るため、実装は以下を満たしてください。

- 列の特定
  - ヘッダ（1 行目付近）から「証券コード」「17 業種コード」「33 業種コード」に該当する列を決定する
  - 見つからない場合は処理を失敗させ、原因が分かるエラーを出す（サイレントに空配列を作らない）
- 行の扱い
  - 行ごとに、以下の値が必要:
    - 証券コード（必須）
    - 17 業種コード（存在する場合に限り使用）
    - 33 業種コード（存在する場合に限り使用）
  - 証券コードが空の行は無視する
- 文字列化
  - 証券コードは常に文字列として扱う
  - 17/33 の業種コードは、先頭ゼロを含む可能性があるため、文字列として扱う
  - 数値として読み込まれてしまう場合は、生成物のキーが文字列として正しく復元されるように実装する
    - 例: `0050` のような先頭ゼロを保持

## 出力マッピング

### `17`（`/api/17/{code}.json`）
- マッピング:
  - `code` = 17 業種コード
  - value = 対応する証券コード集合

### `33`（`/api/33/{code}.json`）
- マッピング:
  - `code` = 33 業種コード
  - value = 対応する証券コード集合

## 重複・ソート
- 重複は排除する（証券コードの集合化）
- 出力配列は安定化のためにソートする
  - 推奨: `string` の辞書順（UTF-8 文字列の昇順）

## 対象範囲
- `data_j.xls` に存在する「17 業種コード」の全てを JSON 化
- `data_j.xls` に存在する「33 業種コード」の全てを JSON 化
