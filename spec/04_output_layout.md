# 4. 出力ファイル構成

GitHub Pages で配信される静的ファイルはリポジトリ内の次のディレクトリに出力します。

## 生成物のルート
- `api/`

## 17 業種コード体系
- `api/17/{code}.json`
  - `{code}` は 17 業種コード（先頭ゼロ保持）
  - 返却: JSON 配列のみ（証券コード文字列の配列）

## 33 業種コード体系
- `api/33/{code}.json`
  - `{code}` は 33 業種コード（先頭ゼロ保持）
  - 返却: JSON 配列のみ（証券コード文字列の配列）

## ファイルの内容（共通）
- エンコーディング: UTF-8
- JSON は次を満たすこと:
  - 配列のみ（オブジェクトでラップしない）
  - 要素は証券コード文字列
  - 重複なし
  - 安定した順序（ソート）

## 例
- `api/17/1.json` -> `["1101","222A","8326"]`
- `api/33/0050.json` -> `["1325","323A","9125"]`
