# 6. 生成時バリデーション（品質チェック）

生成スクリプトは、少なくとも以下を満たすこと。

## 列の検証
- 必要列（証券コード / 17 業種コード / 33 業種コード相当）が見つからない場合は失敗する

## キーと値の検証
- 証券コードは全件「空文字でない」こと
- `api/17/{code}.json` / `api/33/{code}.json` の `{code}` は、入力由来の業種コード文字列と一致していること

## 出力の検証
- 各ファイルの内容が JSON 配列であること（`[` で始まる等の軽い検証でもよい）
- 配列要素に重複がないこと
- 出力配列がソート済みであること（辞書順）
- 生成されたファイル数が、入力に含まれる業種コード数と整合していること（完全一致を推奨）

## ログ
- 生成開始/終了
- 入力ファイルの更新日時やサイズ（可能ならハッシュ）
- 出力した `api/17` / `api/33` のコード数
- 失敗時の原因（どの列が見つからなかった等）
